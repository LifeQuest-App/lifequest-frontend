# Routing and UI Skeleton

## Next.js App Router

Вся навигация построена через `app/`:

- `/` — приветственный экран со ссылками на auth
- `/login` — вход
- `/register` — регистрация
- `/onboarding` — первичная настройка целей
- `/today` — список ежедневных рутины и отметка выполнения
- `/history` — история по дням
- `/profile` — профиль пользователя и цели

Маршруты доступны через `shared/config/routes.ts` (константы ссылок).

## Root layout

`app/layout.tsx` определяет:

- общую тему/стили (через Tailwind и шрифты `next/font`)
- обертку для глобального состояния (через `app/providers.tsx`)
- базовые классы цветов фона/текста

## AppShell (единый каркас)

`shared/ui/app-shell.tsx` отвечает за:

- верхнюю панель навигации
- общую ширину контента
- слоты под основное содержимое страницы

Это помогает сохранять единый UX между экранами “Today / History / Profile”.

## PageHeader (единый стиль заголовка)

`shared/ui/page-header.tsx` использует единые классы для:

- заголовка страницы
- короткого описания под заголовком

## UI-обязательства для будущих фич

При подключении real data и модалок ожидается:

- быстрые действия: “клик → подтверждение”
- empty state: “добавьте первую рутину”
- error state: “не удалось загрузить, повторить”
- доступность: клавиатурная навигация и `aria-label` для интерактивных элементов
