# Profile (Goals and Stats)

## Назначение

Профиль — это:

- идентичность пользователя (имя, аватар)
- цели (например `daily_step_goal`)
- отображение статистики: выполнено за период, текущий streak

## Что именно меняет пользователь

Ожидаемые поля:

- имя пользователя
- аватар (в будущем — загрузка/хранение)
- цели (числовые параметры по активностям)
- настройки напоминаний (boolean/выбор)

## Контракт с backend

Для MVP-потока обычно достаточно:

- чтение профиля: `GET /users/me`
- обновление on-boarding/целей: `PUT /users/onboarding` (или отдельный endpoint для целей)

Точные схемы response/request зависят от доменной модели C# части.

## UI-обязательства

Профиль должен быть:

- быстрым (минимум шагов)
- понятным (видно, какие цели активны)
- доступным (клавиатура/aria для интерактивных элементов)
