# History

## Цель экрана

History нужен, чтобы:

- подтверждать регулярность (“я действительно делаю”)
- дать прозрачную динамику по дням/неделям
- визуализировать streak и выполнение целей

## Форма представления

Поддерживаемый дизайн:

- список по датам или календарь
- на уровне строки (дня) — “какие рутины были выполнены”

## Фильтрация и параметры

Обычно используется query-параметр даты:

- `GET /history?date=YYYY-MM-DD`

UI получает данные и строит:

- выполнено / не выполнено
- базовые графики (на следующей итерации можно подключить Recharts или аналоги)

## Контракт с backend (ожидаемо)

На фронтенде закладывается:

- возможность загрузить completion-факты по выбранной дате
- агрегирование результатов в формат, удобный для календаря/ленты

Ожидаемые endpoint’ы:

- `GET /history?date=...`

## Текущее состояние в коде

Страница `app/history/page.tsx` — UI-каркас.

После интеграции backend:

- подключаем RTK Query query
- заменяем placeholder данных на реальные completions
